Q2
Full Booking Details — 6-Table JOIN
JOIN × 6 passengers · seats flights · airports × 2 payments
▼

Joins bookings ⟕ passengers ⟕ seats ⟕ flights ⟕ airports (aliased a1/a2 for origin & destination) ⟕ payments into one comprehensive result. Classic multi-table relational query demonstrating FK traversal.

SELECT
    b.pnr,
    p.full_name,
    CONCAT(a1.iata_code, ' -> ', a2.iata_code) AS route,
    f.flight_number,
    f.departure_time,
    s.seat_number,
    s.class,
    pay.amount,
    pay.payment_method,
    b.status AS booking_status
FROM bookings b
JOIN passengers p  ON p.passenger_id = b.passenger_id
JOIN seats s       ON s.seat_id      = b.seat_id
JOIN flights f     ON f.flight_id    = s.flight_id
JOIN airports a1   ON a1.airport_id  = f.origin_airport_id
JOIN airports a2   ON a2.airport_id  = f.dest_airport_id
JOIN payments pay  ON pay.booking_id = b.booking_id
ORDER BY b.booking_id;

Q3
Revenue Per Flight — GROUP BY + SUM
GROUP BY SUM() COUNT() ORDER BY DESC
▼

Aggregates total revenue and booking count per flight using GROUP BY. Filters only SUCCESS payments via WHERE, then orders by highest revenue — AI-204 leads with ₹37,200 from 5 confirmed bookings (PNR001 excluded: payment = REFUNDED after Q5).

SELECT
    f.flight_number,
    CONCAT(a1.city, ' -> ', a2.city) AS route,
    COUNT(b.booking_id)              AS total_bookings,
    SUM(pay.amount)                  AS total_revenue
FROM flights f
JOIN airports a1   ON a1.airport_id  = f.origin_airport_id
JOIN airports a2   ON a2.airport_id  = f.dest_airport_id
JOIN seats s       ON s.flight_id    = f.flight_id
JOIN bookings b    ON b.seat_id      = s.seat_id
JOIN payments pay  ON pay.booking_id = b.booking_id
WHERE pay.payment_status = 'SUCCESS'
GROUP BY f.flight_id, f.flight_number, route
ORDER BY total_revenue DESC;

Q4
Business vs Economy Revenue Breakdown
GROUP BY class ENUM column SUM() COUNT()
▼

Groups by the ENUM column seats.class to compare Business vs Economy. Despite fewer bookings (3 vs 4), Business generates ~70% of total revenue — ₹34,800 vs ₹15,100. PNR001 (Economy) was refunded after Q5, reducing Economy count from 5 to 4.

SELECT
    s.class,
    COUNT(b.booking_id) AS bookings,
    SUM(pay.amount)     AS revenue
FROM bookings b
JOIN seats s      ON s.seat_id      = b.seat_id
JOIN payments pay ON pay.booking_id = b.booking_id
WHERE pay.payment_status = 'SUCCESS'
GROUP BY s.class;

Q5
Cancellation Story — Trigger Demo (5 SQL Steps)
UPDATE INSERT ⚡ TRIGGER fires REFUND flow
▼

Full cancellation flow for Priya Sharma (PNR001). The INSERT into cancellations automatically fires trigger restore_seat_on_cancel, flipping seat 4A → is_available = TRUE with no manual UPDATE needed.

-- Step A: Check seat BEFORE cancellation
SELECT seat_number, is_available
FROM seats WHERE seat_id = 4;
-- Result: 4A | FALSE  (BOOKED)

-- Step B: Cancel the booking
UPDATE bookings
SET status = 'CANCELLED'
WHERE pnr = 'PNR001';

-- Step C: Insert cancellation record → TRIGGER FIRES ⚡
INSERT INTO cancellations
    (booking_id, reason, refund_amount, refund_status)
VALUES (1, 'Change of travel plans', 3780.00, 'PENDING');
-- restore_seat_on_cancel: SET seat 4A is_available = TRUE

-- Step D: Mark payment as REFUNDED
UPDATE payments
SET payment_status = 'REFUNDED'
WHERE booking_id = 1;

-- Step E: Verify seat AFTER trigger
SELECT seat_number, is_available
FROM seats WHERE seat_id = 4;
-- Result: 4A | TRUE  (AVAILABLE) ← Trigger worked!

Q6
Flight Occupancy % — LEFT JOIN + ROUND()
LEFT JOIN ROUND() Calculated column NULL → 0 via COUNT
▼

Uses LEFT JOIN to include all 5 flights, even those with 0 bookings. ROUND(COUNT * 100.0 / total_seats, 1) produces occupancy % — COUNT() safely returns 0 for unmatched LEFT JOIN rows (no NULL issues).

SELECT
    f.flight_number,
    CONCAT(a1.city, ' -> ', a2.city)                        AS route,
    ac.total_seats,
    COUNT(b.booking_id)                                     AS booked_seats,
    ROUND(COUNT(b.booking_id) * 100.0 / ac.total_seats, 1) AS occupancy_pct
FROM flights f
JOIN  aircraft ac     ON ac.aircraft_id = f.aircraft_id
JOIN  airports a1     ON a1.airport_id  = f.origin_airport_id
JOIN  airports a2     ON a2.airport_id  = f.dest_airport_id
LEFT JOIN seats s     ON s.flight_id    = f.flight_id
LEFT JOIN bookings b  ON b.seat_id      = s.seat_id
                      AND b.status      = 'CONFIRMED'
GROUP BY f.flight_id, f.flight_number, route, ac.total_seats
ORDER BY occupancy_pct DESC;
